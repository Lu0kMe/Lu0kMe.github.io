SSP Leak

One gadget

内容简介

堆是程序虚拟内存中由低地址向高地址线性增长的线性区域。堆是一块需要动态管理的内存，为了提高管理堆的效率，(Glibc等)实现堆管理的库往往会引入很多机制。不同于栈上的溢出能够直接覆盖函数的返回地址从而劫持程序控制流，堆利用只能通过间接手段来劫持程序控制流。而这些利用手段往往依赖于堆的实现、堆管理机制的实现以及堆的安全检查机制的实现。

TCache机制

发表于 2021-11-28 分类于 Read Notes

简介

在Glibc-2.26版本内存分配器加入了TCache机制而有了较大的变化。TCache全名为Thread Local Caching，它为每个线程创建一个缓存，里面包含了一些小堆块，无须对arena上锁即可使用，这种无锁的分配算法能有不错的性能提升。虽然线程缓存在很多年前已经在另一种内存分配器TCMalloc(Thread-Caching Malloc)中实现，但直到2017年ptmalloc2才在glibc-2.26中将其正式加入，并默认开启。

堆简析

发表于 2021-11-27 分类于 Read Notes

堆简介

堆是程序虚拟内存中由低地址向高地址线性增长的线性区域。一般只有当用户像操作系统申请内存时，这片区域才会被内核分配出来，并且出于效率和页对齐的考虑，通常会分配相当大的连续内存。程序再次申请时便会从这片内存中分配，直到堆空间不能满足时才会再次增长。堆的位置一般在BSS段高地址处。
