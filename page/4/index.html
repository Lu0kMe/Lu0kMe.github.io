SSL协议

发表于 2021-11-07 分类于 Protocol

简介

传输层安全性协议(TLS, Transport Layer Security)及其前身安全套接层(SSL, Secure Sockets Layer)是一种安全协议，目的是为互联网通信提供安全及数据完整性保障。SSL最早的版本是1994年的SSLv1，它是网景公司的内部草案，并未正式公布。后续还有SSLv2、SSLv3以及标准化后的TLS等。SSLv3以及TLS在各种应用程序中广泛使用，已成为互联网保密通信的工业标准。

IPsec协议

发表于 2021-11-06 分类于 Protocol

简介

IPsec(Internet Protocol Security)是IETF(Internet Engineering Task Force)制定的一组开放的网络安全协议。IPsec是一个协议族，其中包含多个协议，包括用于协商协议的互联网安全关联与密钥管理协议(ISAKMP, Internet Security Association and Key Management Protocol)和互联网密钥交换(IKE, Internet Key Exchange)两个交换协议，以及用于数据封装处理的认证首部(AH, Authentication Header)和封装安全载荷(ESP, Encapsulating Security Payload)两个数据封装协议。

发表于 2021-11-01 分类于 Reading Notes

DLL简介

在Windows系统中，动态链接文件被称为动态链接库(DLL, Dynamic-Link Library)，它相当于Linux下的共享对象。Windows系统中大量采用了这种DLL机制，甚至包括Windows的内核的结构都很大程度上依赖于DLL机制。Windows下的DLL文件和EXE文件实际上是一个概念，它们都是有PE格式的二进制文件，稍微有些不同的是PE文件头部中有个符号位表示该文件是EXE或是DLL，而DLL文件的扩展名不一定是.dll，也有可能是别的。

Linux共享库

共享库简介

由于动态链接的诸多优点，大量的程序开始使用动态链接机制，导致系统中存在数量庞大的共享对象。如果没有很好的方法将这些共享对象组织起来，整个系统中的共享对象文件则会散落在各个目录下，给长期的维护、升级造成了很大的问题。操作系统一般会对共享对象的目录组织和使用方法有一定的规则，Linux下使用共享库(Shared Library)的概念来组织共享库。

动态链接简介

静态链接虽然在原理上很简单，但是实现起来却十分困难。一方面，静态链接对于计算机内存和磁盘的空间浪费非常严重；另一方面，静态链接对于程序的更新、部署和发布也会带来很多麻烦。解决这两个问题的方法就是把程序的模块相互分割开来，形成独立的文件，而不再将它们静态地链接在一起。也就是说，把链接的过程推迟到程序运行时再进行，这就是动态链接。
